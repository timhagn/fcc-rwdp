Overview

This Documentation shall introduce the following programming languages and show some of their differences and similarities:

  • JavaScript / ECMAScript
  • PHP

It neither claims nor wants to fulfill completeness, as whole books could be (and were) written about them, each on their own. It just wants to shed light on some interesting concepts similar and different. Where not stated otherwise, it refers to the latest versions of each language.

JavaScript + ECMAScript

JavaScript (JS) is a lightweight interpreted or JIT-compiled programming language with first-class functions. While it is most well-known as the scripting language for Web pages, many non-browser environments also use it, such as Node.js, Apache CouchDB and Adobe Acrobat. JavaScript is a prototype-based, multi-paradigm, dynamic language, supporting object-oriented, imperative, and declarative (e.g. functional programming) styles.

The standard for JavaScript is ECMAScript. As of 2012, all modern browsers fully support ECMAScript 5.1. Older browsers support at least ECMAScript 3. On June 17, 2015, ECMA International published the sixth major version of ECMAScript, which is officially called ECMAScript 2015, and was initially referred to as ECMAScript 6 or ES6. Since then, ECMAScript standards are on yearly release cycles. 

            
/**
 * Minimal JS/ES6 Hello World with Immediately Invoked Function.
 */

(() => {
    const text = `Hello World with ES6!`;
    console.log(text);
})();
PHP

PHP: Hypertext Preprocessor (or simply PHP) is a server-side scripting language designed for Web development, but also used as a general-purpose programming language. It was originally created by Rasmus Lerdorf in 1994, the PHP reference implementation is now produced by The PHP Group. PHP originally stood for Personal Home Page, but it now stands for the recursive acronym PHP: Hypertext Preprocessor.

PHP code may be embedded into HTML code, or it can be used in combination with various web template systems, web content management systems, and web frameworks. PHP code is usually processed by a PHP interpreter implemented as a module in the web server or as a Common Gateway Interface (CGI) executable. The web server combines the results of the interpreted and executed PHP code, which may be any type of data, including images, with the generated web page. PHP code may also be executed with a command-line interface (CLI) and can be used to implement standalone graphical applications.

Python

Python is an interpreted high-level programming language for general-purpose programming. Created by Guido van Rossum and first released in 1991, Python has a design philosophy that emphasizes code readability, notably using significant whitespace. It provides constructs that enable clear programming on both small and large scales.

Python features a dynamic type system and automatic memory management. It supports multiple programming paradigms, including object-oriented, imperative, functional and procedural, and has a large and comprehensive standard library.

Variable Declaration

All three languages share a common similarity in interpreted scripting languages:
Variables are (by default) dynamically typed (with some exception(s ; ) in Python, as you are going to see).
But whereas PHP can be urged to enforce some level of strict typing and JS and it both provide some kinds of defining a "constant" (unchangeable, read-only variables), Python enforces the first with some types and not others and the latter by coding standards, or some "tricks" as shown below (for all of you coming from JS-Land, you might cry "hurray" at inbuilt im-/mutability for some types).

JavaScript

Let's first take a dive into JS, where one can use any Unicode character and only has to conform to certain rules, as taken from MDN:

A JavaScript identifier must start with a letter, underscore (_), or dollar sign ($); subsequent characters can also be digits (0-9). Because JavaScript is case sensitive, letters include the characters "A" through "Z" (uppercase) and the characters "a" through "z" (lowercase). 
            
/**
 * In the "olden days" of pre ES6 there only was the "almighty"
 * var - but with - at the most - a little grumbling it could be
 * let (pun intended) out without "use strict". Example:
 */
var MyVariable = "I'm a string but also might be a Number next";
MyVariable = 1;

/*
 * As of ES6 (ECMAScript 2015) there came:
 * + let - a block-scoped, local variable.
 * + const - a block-scoped, read-only named constant.
 * Don't think about 'scope' right now, the next section will cover this.
 * Examples for let and const (taken from MDN):
 */
function letTest() {
  let x = 1;
  if (true) {
    let x = 2;  // different variable
    console.log(x);  // 2
  }
  console.log(x);  // 1
}

// NOTE: Constants can be declared with uppercase or lowercase,
// but a common convention is to use all-uppercase letters.
// define MY_FAV as a constant and give it the value 7
const MY_FAV = 7;
// this will throw an error - Uncaught TypeError: Assignment to
// constant variable.
MY_FAV = 20;
// MY_FAV is 7
console.log('my favorite number is: ' + MY_FAV);

PHP

Next, let's see how PHP variables and constants can be declared; here, too, some rules have to be met, as taken from php.net:

Variable names follow the same rules as other labels in PHP. A valid variable name starts with a letter or underscore, followed by any number of letters, numbers, or underscores. As a regular expression, it would be expressed thus: '[a-zA-Z_\x7f-\xff][a-zA-Z0-9_\x7f-\xff]*'

Such, in layman's terms: the (extended) ASCII-Standard minus the almighty $, which starts each variable and some special cases like $this. Again they are case sensitive.

            
// First to the basics, taken from PHP.net:
$var = 'Bob';
$Var = 'John';
echo "$var, $Var";      // outputs "Bob, John"

$4site = 'not yet';     // invalid; starts with a number
$_4site = 'not yet';    // valid; starts with an underscore
$täyte = 'mansikka';    // valid; 'ä' is (Extended) ASCII 228.

/*
 * As with 'let' in ES6+ above, variables in PHP are block-scoped by
 * default, too. Variables from the global scope may be accessed by the keyword
 * of the same name. Example:
 */
$var = "I am a global variable.";

function testVars() {
  $var = "I'm a local variable.";
  echo $var;
}

echo $var;              // outputs: "I am a global variable."
testVars();             // outputs: "I'm a local variable."

// Whereas the following:
function testVars() {
  echo $var;            // Will give you a PHP Notice:  Undefined variable: var
}

// Using the 'global' keyword beforehand:
function testVars() {
global $var;
  echo $var;            // outputs: "I am a global variable."
}

Now constants in PHP may be defined in the following ways:

You can define a constant by using the define()-function or by using the const keyword outside a class definition as of PHP 5.3.0. While define() allows a constant to be defined to an arbitrary expression, the const keyword has restrictions as outlined in the next paragraph. Once a constant is defined, it can never be changed or undefined.
When using the const keyword, only scalar data (boolean, integer, float and string) can be contained in constants prior to PHP 5.6. From PHP 5.6 onwards, it is possible to define a constant as a scalar expression, and it is also possible to define an array constant. It is possible to define constants as a resource, but it should be avoided, as it can cause unexpected results.

These are the differences between constants and variables:

  • Constants do not have a dollar sign ($) before them;
  • Prior to PHP 5.3, Constants may only be defined using the define() function, not by simple assignment;
  • Constants may be defined and accessed anywhere without regard to variable scoping rules;
  • Constants may not be redefined or undefined once they have been set; and
  • Constants may only evaluate to scalar values. As of PHP 5.6 it
    is possible to define array constant using const keywords and
    as of PHP 7 array constants can also be defined using define()
    You may use arrays in constant scalar expressions (for example,
    const FOO = array(1,2,3)[0];), but the end result must be a
    value of allowed type.

Examples of defining constants (from PHP.net):

            
define("CONSTANT", "Hello world.");
echo CONSTANT; // outputs "Hello world."
echo Constant; // outputs "Constant" and issues a notice.

Defining Constants using the const keyword:

            
// Works as of PHP 5.3.0
const CONSTANT = 'Hello World';

echo CONSTANT;

// Works as of PHP 5.6.0
const ANOTHER_CONST = CONSTANT.'; Goodbye World';
echo ANOTHER_CONST;

const ANIMALS = array('dog', 'cat', 'bird');
echo ANIMALS[1]; // outputs "cat"

// Works as of PHP 7
define('ANIMALS', array(
    'dog',
    'cat',
    'bird'
));
echo ANIMALS[1]; // outputs "cat"

Python

Now on to Python. IMHO it is both the most beginner friendly of these three mentioned languages, but has is quirks in defining variables, because (from learnpython.org):

Python is completely object oriented, and not "statically typed". You do not need to declare variables before using them, or declare their type. Every variable in Python is an object.

Now you might say: "Ok, what's the difference to JavaScript?", when you hear about objects (as in JS, everything is an object, too, have I forgotten to mention? ; ). But without further ado, let's start with a simple declaration:

            
name = "Guido van Rossum"
print(name)                 # outputs: "Guido van Rossum"

And as easy as that you have declared a String Variable (more on Types in the next chapter). But whereas in JS or PHP, when you redeclare a variable, it get's overwritten, Strings in Python are immutable!
Let's have a look, what you could do in PHP (the same would be possible in JS, too):

            
$name = "Guido van Rossum";
$name[7] = "o";
echo $name;                     // outputs: Guido von Rossum

Trying the same in Python would give you a type error:

            
name = "Guido van Rossum"
name[7] = "o"               # gives you:
'''
Traceback (most recent call last):
  File "input", line 1, in module
TypeError: 'str' object does not support item assignment
'''

And although you seem to be able to redeclare variables in Python, in truth a new object of the same or another type get's created in memory:

            
name = "Guido van Rossum"
name = "Joachim von Hassel" # A new string (str) object get's created!
print(name)                 # outputs "Joachim von Hassel"
# (Off-Topic: Do you know "Welle: Erdball"? ; )

# And of course you can do this:
name = 104                  # but a new integer (int) object get's created!

But as written above, more on types in the next chapter. Now you might begin to understand, that (nearly) no variable is really constant in Python. There are only some workarounds and libraries you can import, but the easiest way might be to use a so called tuple, a special kind of list, but even that's not unbreakable:

            
CONST = ("fixed",)          # The ',' at the end tells Python to define a tuple.
print(CONST[0])             # outputs: "fixed"
CONST[0] = "nada"           # gives you:
'''
Traceback (most recent call last):
  File "input", line 1, in module
TypeError: 'tuple' object does not support item assignment
'''

# There's an inbuilt library "namedtuple", which makes this easier to read:
from collections import namedtuple
Constant = namedtuple('Constant', ['fixed'])
constant = Constant(42)
print(constant.fixed)       # outputs: 42
constant.fixed = 23         # gives you:
'''
Traceback (most recent call last):
  File "input", line 1, in module
AttributeError: can't set attribute
'''

# But you still could do (not writing the CONST[0]):
CONST = "gotcha"
print(CONST)                # outputs: "gotcha" : /

# Respectively:
constant = "gotcha"
print(constant)             # and again outputs: "gotcha" : /

Usually it's just an accepted coding convention to "Just don't change it!", when a variable is written in UPPERCASE.

JS Numeric "Types"

JS / ESx has no "real" (no pun intended) differentiation between the many possible numeric types - although one can find out with some functions of the Number() class, that's especially there for it.
As w3schools states it:

JavaScript has only one type of number. Numbers can be written with or without decimals.

Most of times, that's just fine, just not in some special cases. The ECMAScipt specification talks about "double-precision 64-bit format IEEE 754-2008" floating point numbers. But let's dive straight into code:

                
// Let's start with the MAX_SAFE_INTEGER
console.log(Number.MAX_SAFE_INTEGER);           //outputs: 9007199254740991

// That's an important value to remember (or look up ; ),
// because of JS's _integer_ precision is "only" 15 digits,
// which means an "integer overflow" takes place:
console.log(999999999999999);                   // outputs: 999999999999999
console.log(9999999999999999);                  // outputs: 10000000000000000

// Floating point arithmetic get's up to 17 decimals, but a classic example is:
console.log(0.1 + 0.2);                         // outputs: 0.30000000000000004
// Wouldn't be great in calculating bank transfers, would it ; )?
// To get around this behavior, multiply and divide by 10:
console.log((0.1 * 10 + 0.2 * 10) / 10);        // outputs: 0.3

// One !important thing to remember, is JS's type coercion with concatenation:
console.log("10" + 20);                         // outputs: "1020", not 30!

// And of course JS / ESx is able to handle hexadecimal and other numeric types:
// Hex:
console.log(0xFF);                              // outputs: 255
// Binary:
let giveMeBinary = 32;
console.log(giveMeBinary.toString(2));          // outputs: 100000
// ...whereas with Number.toString() you can output numbers from base 2 (binary)
// to base 36. Hexadecimal is base 16. Decimal is base 10. Octal is base 8.

PHP Number types

As in JS, PHP has many different numeric types, but no real differentiation. But usually they are divided up in Integers and Floating point numbers. First some code for integer Types, as they include decimal (base 10), hexadecimal (base 16), octal (base 8) and binary (base 2) notations (taken from "Integers" on php.net:

                
$a = 1234;                  // decimal number
$a = -123;                  // a negative number
$a = 0123;                  // octal number (equivalent to 83 decimal)
$a = 0x1A;                  // hexadecimal number (equivalent to 26 decimal)
$a = 0b11111111;            // binary number (equivalent to 255 decimal)

And as in JS there are some borders to MAX and MIN safe integers, declared in PHP constants as follows - but hugely depending on your type of System (e.g. 32 or 64 bit) and plattform (as on Windows prior to PHP 7, they always were 32 bit):

                
/*
 * My system is running PHP 7.2 on 64 bit Linux
 */

// The maximum Integer size in byte is:
echo PHP_INT_SIZE;                      // outputs: 8

// which leads to a maximum integer Number of:
echo PHP_INT_MAX;                       // outputs: 9223372036854775807

// and a minimum integer Number of:
echo PHP_INT_MIN;                       // outputs: -9223372036854775808

/*
 * As in JS, integer overflows with very large numbers may occur
 * - but whereas they get simply "rounded up" in JS, in PHP they
 * get converted to a floating point number (or simply float):
 */
var_dump(PHP_INT_MAX + 1);              // outputs: float 9.2233720368548E+18

Which was a great way to switch to floating point numbers in PHP, which have the same precision as in JS (IEEE 754 double precision format) or as php.net warns:

Floating point numbers have limited precision. Although it depends on the system, PHP typically uses the IEEE 754 double precision format, which will give a maximum relative error due to rounding in the order of 1.11e-16. Non elementary arithmetic operations may give larger errors, and, of course, error propagation must be considered when several operations are compounded. [...] So never trust floating number results to the last digit, and do not compare floating point numbers directly for equality.

But let some code speak for itself:

                
// Floating point numbers (float or "real numbers") can be specified as such:
$standard    = 1.234;
var_dump($standard);                    // outputs: float 1.234

$exponential = 1.2e3;
var_dump($exponential);                 // outputs: float 1200

$scientific  = 7E-10;
var_dump($scientific);                  // outputs: float 7.0E-10
// Or easier understandable:
$positive_scientitific = 7E10;
var_dump($positive_scientitific);       // outputs: float 70000000000

// And adhere to the warning above, be careful with comparing floats:
$a = 40 - 17.0;
var_dump($a);                           // outputs: float 23
$b = 23.0;
var_dump($b);                           // outputs: float 23
var_dump($a == $b);                     // outputs: boolean true

// That was as "expected". Whereas:
$x = 60 - 17.77;
var_dump($x);                           // outputs: float 42.23
$y = 42.23;
var_dump($y);                           // outputs: float 42.23
var_dump($x == $y);                     // outputs: boolean false

Since PHP 7.2.0 there are some global constants that provide some insights into the "depths" of floating point numbers, as with integers above:

                
// Number of decimal digits that can be rounded into a
// float and back without precision loss.
echo PHP_FLOAT_DIG;                     // outputs: 15
// Smallest representable floating point number.
echo PHP_FLOAT_MIN;                     // outputs: 2.2250738585072E-308
// Largest representable floating point number.
echo PHP_FLOAT_MAX;                     // outputs: 1.7976931348623E+308
// Smallest representable positive number x, so that x + 1.0 != 1.0.
echo PHP_FLOAT_EPSILON;                 // outputs: 2.2204460492503E-16
